git init Inicia o git e cria um repositório dentro do diretório atual

Uma alteração no meu repositório local (ambiente de desenvolvimento) não altera o repositório que está no servidor REMOTE REPOSITORY(github) mas existem comandos específicos que façam isso.

Os arquivos sempre ficam transitando entre working directory e staging área mas a partir do momento que eu uso o comando commit ele pega tudo que tava na staging area e se torna um commit junta as informações e depois tira uma “foto” e os arquivos voltam par working directory 
tudo que está no local repository tem que estar dentro do commit pois se não, não teria como enviar os repositórios locais para um repositorio remoto(GitHub)
